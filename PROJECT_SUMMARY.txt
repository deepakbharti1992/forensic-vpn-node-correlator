----------------------------------------------------------------
  MERI REQUIREMENT (USER REQUIREMENTS)
----------------------------------------------------------------

1. EK AISA TOOL CHAHIYE THA JO:
   - Android phone pe jo VPN (NordVPN) chal raha hai uska
     server IP automatically capture kare
   - Jab bhi VPN reconnect ho to naya IP record ho jaye
   - Desktop pe packet capture karke VPN traffic dekhe
   - Entry node (phone se dikha IP) aur Exit node
     (actual captured packet ka IP) ko correlate kare
   - Sabhi data forensic evidence ke liye CSV aur Excel
     mein save ho

2. ANDROID APP CHAHIYE THI JO:
   - Background mein chal sake (foreground service)
   - NordVPN ke UI se IP automatically padh sake
   - Desktop se TCP connection maintain kare
   - Jab desktop bole tab VPN reconnect kare

3. DESKTOP APP CHAHIYE THA JO:
   - Live packet capture kare hotspot NIC pe
   - Phone se data receive kare TCP ke zariye
   - VPN entry aur exit nodes ko match kare
   - SHA-256 hash ke saath tamper-proof CSV banaye
   - Excel export bhi kare

4. END-TO-END AUTOMATION CHAHIYE THI:
   - ADB se phone control ho sake
   - NordVPN automatically disconnect/reconnect ho
   - Naya IP automatically record ho
   - Lagatar (continuous) monitoring ho sake

5. GITHUB PE CODE SAVE HO:
   - Public repository
   - README, License, Description, Topics sab ho

----------------------------------------------------------------
  HAMNE KYA KIYA (IMPLEMENTATION)
----------------------------------------------------------------

PHASE 1 — DESKTOP PYTHON APPLICATION
--------------------------------------
Banaye gaye files:
  - capture_engine.py   : Scapy AsyncSniffer se real-time
                          packet capture, 12-second rolling
                          buffer, phone IP filter
  - vpn_detector.py     : VPN traffic pehchanne ke liye
                          port heuristics (WireGuard:51820,
                          OpenVPN:1194, etc.), known NordVPN
                          IP ranges
  - pairing_engine.py   : Entry node + Exit node correlation
                          algorithm (3-tier priority, ±3s
                          time window, /24 subnet matching)
  - mobile_bridge.py    : TCP server port 5000, JSON
                          framing, PING/PONG heartbeat
  - csv_exporter.py     : SHA-256 row hashing, append-only
                          CSV, Excel export (openpyxl)
  - gui_dashboard.py    : PyQt6 live dashboard, confidence
                          bar, mobile connection status
  - main.py             : Application entry point
  - demo_runner.py      : Demo mode with sample data

PHASE 2 — ANDROID APK (KOTLIN)
---------------------------------
Banaye gaye files:
  - TcpClientService.kt           : Persistent TCP client,
                                    auto-reconnect, PING/PONG
                                    response, send queue
  - ForensicAccessibilityService.kt: NordVPN UI se IP padhna,
                                    reconnect button tap karna
  - IpExtractor.kt                : Accessibility tree se
                                    public IPv4 extract karna
  - MainActivity.kt               : App UI, service controls
  - AndroidManifest.xml           : Permissions, services

APK build: VNCForensic-debug.apk (3.03 MB)
Phone pe install: Samsung Galaxy Note 20 Ultra

PHASE 3 — SETUP & CONFIGURATION
----------------------------------
  - Android SDK download (platform-tools, build-tools)
  - Npcap install (Windows packet capture ke liye)
  - Windows Hotspot configure (192.168.137.1)
  - Phone ko hotspot se connect (192.168.137.184)
  - ADB USB debugging enable
  - AccessibilityService enable via ADB
  - Windows Firewall rule add (port 5000)
  - ADB reverse tunnel setup (USB se VPN bypass)

----------------------------------------------------------------
  IMPORTANT BUGS FIXED DURING DEVELOPMENT
----------------------------------------------------------------

1. Android threading deadlock:
   TcpClientService mein single executor se PONG kabhi
   nahi ja raha tha. Fix: alag LinkedBlockingQueue +
   dedicated send thread banaya.

2. ADB reverse tunnel:
   NordVPN ka VPN sab traffic route karta tha, isliye
   192.168.137.1:5000 connect nahi hota tha.
   Fix: adb reverse tcp:5000 tcp:5000 (USB tunnel).

3. Timing window miss:
   30s polling ke baad event inject karne pe 12s buffer
   expire ho jata tha.
   Fix: force-stop ke saath reconnect_time turant set.

4. NordVPN UI tap intercept:
   Jetpack Compose map view sab taps intercept karta tha.
   Fix: force-stop → relaunch approach.

5. Windows Firewall:
   Port 5000 block tha. Fix: netsh firewall rule add kiya.
